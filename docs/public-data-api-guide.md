# 공공데이터 포털 식당 API 가이드

## 🎯 공공데이터 장단점

### ✅ 장점
- **완전 무료** (무제한 사용)
- **합법적** (정부 공식 데이터)
- **법적 위험 제로**
- **회원가입만으로 사용 가능**

### ❌ 단점
- **업데이트 느림** (월 1회 또는 분기별)
- **사용자 경험 데이터 없음** (평점, 리뷰 등)
- **폐업 정보 반영 느림**
- **좌표 정확도 낮을 수 있음**
- **API 응답 속도 느림**

---

## 📋 주요 식당 관련 공공데이터 API

### 1. 식품의약품안전처 - 일반음식점 인허가 정보 ⭐ 추천

**데이터:**
- 업소명, 주소, 전화번호
- 업태, 업종
- 인허가 날짜
- 폐업 여부

**장점:**
- 전국 모든 음식점 포함
- 공식 인허가 데이터

**단점:**
- 평점, 리뷰 없음
- 좌표 없음 (주소만 제공)
- 영업시간, 메뉴 정보 없음

**API 주소:**
```
https://www.foodsafetykorea.go.kr/api/{인증키}/COOKSERVICE/json/1/100/SIGUN_NM={구로구}
```

---

### 2. 서울시 열린데이터광장 - 일반음식점 인허가 정보

**데이터:**
- 서울시 음식점만
- 업소명, 주소, 전화번호
- 업태, 좌표(X, Y)
- 인허가 일자

**장점:**
- 좌표 포함!
- 서울시만 대상이라 정확도 높음

**단점:**
- 서울시만 가능
- 업데이트 느림

**API 주소:**
```
http://openapi.seoul.go.kr:8088/{인증키}/json/LOCALDATA_072404/1/1000/
```

---

### 3. 한국관광공사 - 음식점 정보 (Tour API)

**데이터:**
- 관광지 주변 맛집
- 좌표, 설명, 이미지
- 영업시간, 메뉴

**장점:**
- 콘텐츠 풍부
- 좌표 포함
- 이미지 제공

**단점:**
- 유명 맛집만 (전체 음식점 아님)
- 데이터 적음

**API 주소:**
```
http://apis.data.go.kr/B551011/KorService1/searchKeyword1
```

## 🚀 발급 방법

### 1단계: 회원가입
https://www.data.go.kr/ 접속 → 회원가입

### 2단계: 활용 신청
1. 원하는 API 검색 (예: "음식점")
2. "활용신청" 클릭
3. 사유: "지역 맛집 지도 서비스 개발"
4. 승인 대기 (즉시 ~ 1일)

### 3단계: 인증키 발급
- 마이페이지 → 인증키 확인
- 일반인증키 또는 서비스키

## 💡 현실적인 추천

### 🎯 최적 조합: 공공데이터 + 카카오/네이버

```
공공데이터: 기본 식당 정보 (이름, 주소, 전화번호)
↓
카카오/네이버 API: 좌표 보정 + 최신 정보
↓
사용자 리뷰: 평점, 리뷰, 추천
```

### 예시 워크플로우:

1. **초기 데이터**: 공공데이터에서 구로구 음식점 전체 수집
2. **좌표 변환**: 카카오 주소검색 API로 주소 → 좌표
3. **검증**: 실제 영업중인지 네이버/카카오로 확인
4. **사용자 데이터**: 직장인들이 평점, 리뷰 작성

## 🎯 구디 맛집 프로젝트 추천 전략

### Option A: 빠른 MVP (추천!)
```
카카오/네이버 API만 사용
→ 5분이면 시작 가능
→ 사용자가 직접 등록
```

### Option B: 완벽한 데이터베이스
```
1. 공공데이터로 전체 음식점 수집
2. 카카오 API로 좌표 보정
3. 사용자 리뷰로 큐레이션
→ 시간 오래 걸림 (1주일+)
```

### Option C: 하이브리드 (현실적)
```
1. 초기: 카카오/네이버 API 사용
2. 사용자 증가 후: 공공데이터 추가
3. 장기: 자체 DB 구축
```

**지금 단계에서는 공공데이터 비추천!**

**이유:**
1. 발급 1일 대기
2. 좌표 변환 추가 작업 필요

- 카카오 API (5분 발급) → 바로 시작
